## Conflict Report
Trong mô phỏng xung đột Many-to-One, các thành viên cùng chỉnh sửa một file và lần lượt merge vào nhánh main. Một thành viên đã merge trước, dẫn đến xung đột khi các thành viên còn lại thực hiện merge sau đó.

Để xử lý, các thành viên bị xung đột đã pull code mới nhất từ nhánh main về nhánh của mình và tiến hành resolve conflict thủ công bằng cách xem xét, chỉnh sửa và kết hợp các đoạn code bị trùng. Phương pháp được sử dụng là Local Merge Resolution.

## Value of Code Review
Code Review giúp phát hiện lỗi sớm, cải thiện chất lượng code và đảm bảo các tiêu chuẩn chung của nhóm được tuân thủ. Ngoài ra, việc review code còn giúp các thành viên học hỏi lẫn nhau, chia sẻ kiến thức và giảm rủi ro khi merge code vào nhánh chính.

## Agile Events Reflection
Các sự kiện Agile như mini-Daily Standup giúp nhóm nắm được tiến độ công việc, chia sẻ khó khăn và thống nhất kế hoạch tiếp theo. Nhờ trao đổi thường xuyên, nhóm có thể xử lý vấn đề kịp thời và giữ cho dự án đi đúng hướng.
